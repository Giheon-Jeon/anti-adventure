# 2주차 학습 내용: 한글 인코딩과 문자열 처리

## 1. 한글 인코딩 문제 이해 (Character Encoding)

### 1.1 현상
- 윈도우 환경에서 C 언어로 프로그램을 작성하고 실행했을 때, `printf`로 출력한 한글이 `罹먮┃???앹꽦`과 같이 외계어로 표시되는 현상.

### 1.2 원인: 인코딩 불일치 (Encoding Mismatch)
- **소스 코드 인코딩 (UTF-8):** 최근 대부분의 코드 에디터(VS Code 등)는 문자를 저장할 때 전 세계 공통 표준인 `UTF-8` 방식을 사용합니다.
- **터미널 인코딩 (CP949):** 윈도우의 기본 한글 터미널(CMD/PowerShell)은 과거 표준인 `CP949(EUC-KR 확장)` 방식을 기대합니다.
- **결과:** 터미널은 소스 코드의 `UTF-8` 데이터를 `CP949` 방식으로 해석하려고 시도하며, 이 과정에서 한글이 깨지게 됩니다.

### 1.3 해결 방법: 터미널 코드 페이지 변경
프로그램이 실행될 때 터미널의 인코딩 방식을 강제로 `UTF-8(준거 번호 65001)`로 맞추어 해결할 수 있습니다.

```c
#include <stdlib.h> // system() 함수 사용을 위해 필요

int main() {
    // 터미널 인코딩을 UTF-8로 변경 (Windows)
    system("chcp 65001 > nul"); 
    
    // 이후의 printf("한글")은 정상 출력됨
}
```

## 2. Pro-Tips 💡
- **`.shape` 확인 습관:** 데이터 분석이나 딥러닝 시 텐서나 배열의 차원을 확인하듯, C 언어에서는 문자열의 끝에 항상 `\0` (NULL 문자)가 포함되어 공간을 1바이트 더 차지한다는 점을 기억하세요!
- **인코딩 설정 확인:** VS Code 하단 상태 표시줄에서 `UTF-8`인지 확인하는 습관을 들이면 좋습니다.

### 오답 노트 & 트러블슈팅
- **Q:** `system("chcp 65001")`을 했는데 화면에 "Active code page: 65001"이라는 메시지가 떠요.
- **A:** 코드 뒤에 `> nul`을 붙여주면 해당 메시지를 숨기고 조용히 인코딩만 바꿀 수 있습니다.
