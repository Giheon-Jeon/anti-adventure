# 4주차: TRPG 직업 시스템 확장 및 성장 제어

## 1. 확장된 직업 시스템 (Job Advancement)
- **개요**: 캐릭터의 레벨이 10에 도달하면, 당시의 능력치 분포에 따라 자동으로 전직이 결정됩니다.
- **직업 분류**:
  - **Single (4종)**: 전사(STR), 궁수(DEX), 마법사(INT), 도적(LUK)
  - **Hybrid (6종)**: 두 능력치가 동률일 때 (검투사, 성기사, 광전사, 레인저, 암살자, 현자)
  - **Elite (4종)**: 세 능력치가 동률일 때 (초인, 심판자, 성궤사, 대마법사)
  - **Legendary (1종)**: 모든 능력치가 동률일 때 (아바타)

## 2. 무작위 능력치 성장 (Random Stat Growth)
- **전직 전**: 4개 능력치(`STR`, `DEX`, `INT`, `LUK`)에 25% 확률로 각각 배분 (총 5포인트).
- **전직 후**: 해당 직업의 '주력 능력치'들 사이에서만 무작위로 배분되어 각 직업의 전문성을 높임.
- **의도**: 플레이어에게 매번 다른 육성 경험을 제공하고, 특정 직업을 갖기 위한 수동 조작을 배제함.

## 3. 직업별 특수 스킬 (Dice Multipliers)
- 각 직업은 특정 주사위 족보(`Yacht`, `Straight`, `Triple` 등)가 나왔을 때 고유의 데미지 배율을 적용받습니다.
- **예시**:
  - **도적/암살자**: `Yacht` 발동 시 `Deadly Luck` (1000배 / 300배)
  - **궁수/레인저**: `Straight` 계열 강화
  - **전사**: `Triple` 강화 (파이널 어택)

## 4. 활동 기반 스탯 보정 (Activity Events)
- 무작위 성장 속에서도 플레이어가 원하는 방향으로 조금씩 유도할 수 있도록 필드 이벤트를 추가했습니다.
- **훈련장**: `STR` 또는 `DEX` 직접 강화.
- **도서관**: `INT` 강화 및 경험치 획득.
- **뒷골목 도박**: `LUK` 강화 (실패 시 골드 손실).

## 5. 트러블슈팅 및 오답 노트
- **분배 로직 오류**: `rand() % 4`를 이용한 분배 시 특정 스탯에 치우치는 느낌이 들 수 있으나, 표본이 쌓이면 확률적으로 균등해집니다.
- **전직 조건**: 정확히 '동률'인 경우를 판정하기 위해 `max` 값 확인 후 `is_s, is_d...` 플래그를 활용하는 로직이 효율적입니다.
- **컴파일 경고**: 함수 순서(Function Prototype)나 선언되지 않은 함수 호출에 주의해야 합니다 (`event.c` 수정 기록).

## 💡 Pro-Tips!
- **.shape 확인 습관**: (AI 과정 연계) 데이터 분석 시 Tensor의 차원을 찍어보듯, TRPG에서도 `Player` 구조체의 각 멤버 변수(`.hp`, `.str` 등)를 수시로 `printf`로 디버깅하여 의도한 대로 계산이 흐르는지 확인하세요.
- **Pythonic Tip**: C에서는 복잡한 분기(`if-else`)가 많지만, Python이라면 Dictionary나 List Comprehension으로 더 짧게 표현할 수 있었을 것입니다.
